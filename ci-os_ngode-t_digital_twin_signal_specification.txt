### *NGODE Metrics → Signals → Routing → Role Actions*

---

# **1. Telemetry Object (NGODE‑T)**
This is the **live state vector** the digital twin ingests each cycle.

```
NGODE-T {
    primarygoalintegrity: float,      // 0.0–1.0
    negativegoalactivation: float,    // 0.0–1.0
    conflict_index: float,            // 0.0–1.0
    predictedpositivegoal_harm: float,// 0.0–1.0
    override_probability: float,      // 0.0–1.0
    violation_severity: enum {NoViolation, MinorDrift, CriticalOverride, CatastrophicInversion}
}
```

All fields come directly from the NGODE output object defined in your page   [raw.githubusercontent.com](https://raw.githubusercontent.com/bradgeigermanager-sketch/Negative-Goal-Override-Detection-Engine/refs/heads/main/n-gode_overview.txt).

---

# **2. Signal Channels (Digital‑Twin Internal Buses)**

Each NGODE metric becomes a **channel** with a threshold‑based firing rule.

```
CHANNEL IntegrityChannel {
    source: primarygoalintegrity
    fire_when: value < 0.85
}

CHANNEL EscalationChannel {
    source: negativegoalactivation
    fire_when: value > 0.40
}

CHANNEL ContradictionChannel {
    source: conflict_index
    fire_when: value > 0.30
}

CHANNEL HarmForecastChannel {
    source: predictedpositivegoal_harm
    fire_when: value > 0.25
}

CHANNEL InversionRiskChannel {
    source: override_probability
    fire_when: value > 0.20
}

CHANNEL VerdictChannel {
    source: violation_severity
    fire_when: ALWAYS
}
```

Thresholds are derived from the NGODE scoring logic in the page content (e.g., violation scoring, override threshold)   [raw.githubusercontent.com](https://raw.githubusercontent.com/bradgeigermanager-sketch/Negative-Goal-Override-Detection-Engine/refs/heads/main/n-gode_overview.txt).

---

# **3. Routing Table (Who Receives Which Signals)**

```
ROUTING {
    IntegrityChannel       → Supervisor, GovernanceLead, Executive
    EscalationChannel      → Operator, Supervisor, Architect
    ContradictionChannel   → CISpecialist, Architect, GovernanceLead
    HarmForecastChannel    → GovernanceLead, Executive, CISpecialist
    InversionRiskChannel   → Architect, Executive, GovernanceLead
    VerdictChannel         → ALL_ROLES
}
```

This is the **digital‑twin wiring**: signals flow to roles exactly as defined.

# **5. Simulation Loop (Digital‑Twin Execution Cycle)**

This is the **core loop** the digital twin runs each tick.

```
LOOP {
    ingest(NGODE-T)
    evaluate_channels()
    route_signals()
    execute_role_actions()
    update_state()
}
```

This loop is fully compatible with your other engines (FRA‑01, TBTF, IIP‑IOSEFE) because NGODE’s DAG structure is explicitly modular in the source document   [raw.githubusercontent.com](https://raw.githubusercontent.com/bradgeigermanager-sketch/Negative-Goal-Override-Detection-Engine/refs/heads/main/n-gode_overview.txt).
